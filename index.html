Etiquetas de Texto

Poseemos distintas etiquetas de texto. Algunas de ellas son:

<b> - para Negrita, se utiliza para resaltar texto.

<i> - para Itálica, se utiliza para dar énfasis a un texto.

<u> - para Subrayar, se utiliza para dar énfasis a un texto.

<strong> - para resaltar y dar Importancia a un texto.

<h1> <h2> <h3> <h4> <h5> <h6> - Son distintos tipos de cabecera de texto, siendo H1 la mas importante, y H6 la de menor importancia.

Centrar un DIV

Se sabe que centrar un div sin utilizar Flexbox puede ser una tarea muy difícilo para muchos desarrolladores y diseñadores. A continuación mostraremos algunas maneras de centrar elementos vertical y horizontalmente.

Centrar Horizontalmente

Utilizando Margen Automático: 
<div style="max-width:25%;margin: 0 auto;">Centered</div>
María Laura
Utilizando Inline-block: 
<div style="text-align:center;">
  <div style="display: inline-block;">Centered</div>
</div>
María Laura

Centrado Vertical

<div style="vertical-align:middle;display:inline-block;">
María Laura

Tablas

Podemos crear tablas en HTML y estilarlas con CSS. Los tags de las tablas se agrupan en cabeceras, contenido y pie. Se declaran primero las filas, y luego las celdas separadas en columnas.

Tag Descripción
<table> Define una Tabla
<thead> Agrupa el contenido de cabeceras de la tabla
<tbody> Agrupa el contenido del cuerpo de la tabla
<tfoot> Agrupa el contenido del Pié de la tabla
<th> Define una Celda como cabecera
<tr> Define una Fila de la tabla
<td> Define una Celda de la tabla
Atributo colspan Define el ancho de una celda en cantidad de celdas a ocupar: <td colspan="2" > ocupara 2 celdas de ancho.
Otros atributos Se recomienda NO utilizar atributos estéticos de la tabla, en vez utilizar sus alternativas en CSS. Algunos de los atributos a evitar son: cellpadding, bgcolor, border, cellspacing, width

Layout Tags

Las Tags de Disposición/Layout nos permiten acomodar nuestros elementos en la web. Segmentando nuestrá página en partes, es mas fácil identificar y desarrollar la misma.

Etiqueta Description
<header> Defines a header section on a page or a section of a page
<main> Creates a container for the main content of the page
<footer> Defines a footer section on a page or a section of a page
<nav> Creates a container for navigation links
<aside> Creates a content area that is related to the primary content on a page
<article> Defines a container for independent and self contained text
<section> Defines a piece of content that can stand on its own
<div> Creates a division or section on a page

Colores

Agregar color a su texto HTML es fácil! En este breve tutorial veremos cómo cambiar el color de tu texto HTML con color en Hex los códigos, nombres de colores HTML, RGB y HSL valores.

Color del texto con códigos de color HEX

La forma más común de texto HTML coloración es mediante el uso de códigos de color hexadecimal (código Hex para abreviar). Sólo tienes que añadir un atributo de estilo al elemento de texto que desea de color - un párrafo en el siguiente ejemplo - y el uso de la propiedad de color con su codificación Hex.

                        
                            <p style="color:#FF0000;">Red paragraph text</p>

Color del texto con nombres de colores HTML

Otra forma de color texto de su sitio web es mediante un nombre de color HTML. El código HTML es similar, simplemente reemplace el código Hex en el paso anterior con el nombre del color que desee utilizar (rojo en nuestro ejemplo). Hay 140 colores para elegir.

                        
                            <p style="color:blue;">Blue paragraph text</p>

Color del texto utilizando los valores de los colores RGB

Usando valores RGB para algunos genera rabia, pero es tan fácil como los códigos Hex o los nombres de los colores. Introduzca los valores de RGB en el rgb() parámetro después la propiedad color.

                        
                            <p style="color:rgb(0,255,0);">Green paragraph text</p>

Cuando se utiliza un valor RGB en su sitio web, también puede especificar la opacidad. En lugar de rgb() utilizar rgba() - la a es para alfa, el canal del color que controla la opacidad, y después de su tres valores de color añadir un cuarto de opacidad, en una escala de 0 - 1 (0 para totalmente transparente, 1 para completamente opaco).

                        
                            <p style="color:rgba(0,0,0,0.5);">Opacity 50% of Red</p>

Color del texto utilizando los valores de los colores HSL (Hue, Saturation, Lightness)

Un cuarto método para añadir color HSL es mediante valores. Similar a la sintaxis que se describe arriba RGB, HSL utiliza la hsl() prefijo y tres valores de tono, saturación y luminosidad. El tono es representado en una escala de 0 a 360, mientras que saturación y luminosidad son un porcentaje entre el 0% y el 100%.

                        
                            <p style="color:hsl(0,100%,50%);">Red paragraph text</p>

Y como con RGB también se le puede agregar un 4to parámetro para la opacidad utilizando hsla()

                        
                            <p style="color:hsla(0,100%,50%, 0.5);">Red paragraph text</p>
En conclusión, hay muchas maneras de mostrar los colores en nuestra web, y dependiendo el caso podremos utilizar el que prefiramos.

Flexbox

Antes del módulo Flexbox Layout, había cuatro modos de diseño:

  • Block, para secciones de una página web
  • Inline, para texto
  • Table, para datos de tablas bidimensionales
  • Flotacion/Posicionamiento, para la posición explícita de un elemento
El módulo de diseño de FlexBox facilita el diseño de una estructura de diseño flexible y responsiva sin utilizar flotación ni posicionamiento.

Juego para probar tus habilidades de FlexBox: FlexBox Froggy

Flex-Direction

Dirección de la fila. Define cual va a ser el eje principal para otras propiedades como justify-content/algin items.

                        
    .container {
        display: flex;
        flex-direction: row | row-reverse | column | column-reverse;
    }

Flex-wrap

Reordena los elementos llevandolos a la sigueinte linea en caso de super el espacio en la linea. 

                            
    .container {
        flex-wrap: nowrap; /* por defecto, no realiza Wrap */
        flex-wrap: wrap; /* Activa funcionalida de Wrap */
        flex-wrap: wrap-reverse; /* Invierte el orden del Wrap */
    }

Justify-Content

Ordena los elementos en el eje horizontal (con flex-direction: row) o vertical (con flex-direction: column)

                        
    .container {
        justify-content: flex-start; /* por defecto, alinea los elementos a la izquierda (o arriba en flex-direction: column) */
        justify-content: flex-end; /* alinea los elementos a la derecha (o abajo en flex-direction: column)*/
        justify-content: center; /* alinea los elementos en el centro de la fila / columna */
        justify-content: space-between; /* agrega el máximo espacio posible entre elementos  */
        justify-content: space-around; /* agrega espacio entre elementos y a los lados  */
        justify-content: space-evenly; /* agrega el mismo espacio entre elementos y a los lados  */
    }

Align-Items

Ordena los elementos en su eje vertical

                        
    .container {
        align-items: flex-start; /* por defecto. Alinea los elementos en la parte superior de la caja flex */
        align-items: flex-end; /* alinea los elementos en la parte inferior de la caja flex */
        align-items: center; /* alinea los elementos en el centro de la caja */
        align-items: stretch; /* extiende los elementos para que ocupen el mayor espacio posible verticalmente  */
        align-items: baseline; /* alinea los elementos centrando su texto central  */
    }

Align-self

Permite modificar individualmente el comportamiento de uno o más de los elementos hijos del contenedor flex cambiando su comportamiento previamente definido.

Flex-grow

Especifica el espacio que ocuparan los elementos respecto a sus hermanos en el mismo contenedor. Por defecto es 0, y si se define un valor, el espacio restante se repartirá entre los elementos de manera proporcional a su valor de flex-grow.

                            
    /* Ejemplo 1 */
    #item1 { flex-grow: 1; /* ocupa 1/3 del espacio restante */ }
    #item2 { flex-grow: 1; /* ocupa 1/3 del espacio restante */ }
    #item3 { flex-grow: 1; /* ocupa 1/3 del espacio restante */ }

    /* Ejemplo 2 */
    #item1 { flex-grow: 1; /* ocupa 1/4 del espacio restante */ }
    #item2 { flex-grow: 2; /* ocupa 2/4 del espacio restante */ }
    #item3 { flex-grow: 1; /* ocupa 1/4 del espacio restante */ }

Align-content

En contenedores multilinea especifica la distribución de los elementos Flex. No tiene influencia en los elementos de una única fila.

Selectores mediante ID y Clase

  • Selector universal: *
  • Selector de elementos por etiqueta, puede ser cualquier elemento HTML, ejemplo: div | img | a | h3
  • Selector de elementos por ID: #id-elemento
  • Selector de elementos por clases: .nombre-clase
  • Selectores compuestos: p.prueba selecciona SOLO los párrafos que tengan la clase prueba ignorando los otros elementos (h2, div, etc...) que puedan tenerla.
  • Selector jerárquico, indica qué afectará al elemento más a la derecha, siempre que se cumpla la jerarquía de la izquierda, en el ejemplo esto afectará a los párrafos dentro de un elemento con la clase “prueba”: .prueba p
  • Estos pueden tener todos los niveles necesarios: .prueba p a (afectará a los enlaces que estén dentro de párrafos, siempre que estos estén dentro de un elemento con la clase “prueba”).

Pseudo-selectores

Pseudo-selectores, son selectores que permiten hacer selecciones más precisas, basadas en estados, posiciones y otras características, los reconocemos porque pueden ser casi cualquiera de los selectores arriba mencionados seguidos (sin espacio) de dos puntos ":" y el pseudo-selector.

  • p:first-child selecciona todos los párrafos que sean el primer hijo de un elemento.
  • article:last-child selecciona todos los elementos article que sean el último hijo de un elemento.
  • li:nth-child(n) selecciona todos los li que sean el n hijo de un elemento, este pseudo-selector tiene algunas posibilidades especiales como:
    • section:nth-child(odd) selecciona todas las secciones que sean hijas impares (1,3,5,...) de un elemento.
    • section:nth-child(even) selecciona todas las secciones que sean hijas pares (2,4,6,...) de un elemento.
  • p:nth-of-type(2) selecciona el segundo hijo del tipo párrafo.
  • li:nth-child(3n+2) permite seleccionar utilizando una fórmula (xn+offset) en donde x es el intervalo y offset es el desfase (comenzando en 1), en el ejemplo quedarían seleccionados cada tres li empezando por el segundo(2,5,8,11,...).
  • a:hover selecciona todos los enlaces siempre y cuando el cursor esté sobre ellos (mouse over).

Ejemplo de examen:

Del siguiente código interpretar las siguiente reglas CSS y piensa como quedará cada item:

                        section h1:first-child { color: green; }
                        section *:first-child { color: yellow; }
                        section p:nth-child(2) { color: red; }
                        section p:nth-of-type(2) { color: blue; }

                        <section>
                            <span>Primer Hijo</span>
                            <h1>Segundo Hijo</h1>
                            <p>Tercer Hijo</p>
                            <p>Cuarto Hijo</p>
                        </section>

Se pueden utilizar varios selectores en la misma regla, separándolos con coma (,) ejemplo:

            h1,h3,p{
                /* esta regla afectará a los h1, h3 y p */
            }

También se pueden combinar para crear selecciones muy específicas, ejemplo:

            article p.prueba:last-child{
                /* esta regla afectará a todos los párrafos con la clase prueba, que además sean el últi-
                mo hijo de un elemento article */
            }

Pseudo-Elementos

Son elementos inline que podemos crear desde el CSS para que se añadan antes o después del contenido de un elemento, los reconocemos porque después del selector tienen un par de dos puntos (::) ejemplo:

  • li::before crea un elemento antes del contenido de todos los li
  • li::after crea un elemento después del contenido de todos los li

En ambos casos, la regla necesita la propiedad content, en la cual va el “contenido” del pseudo-elemento. Estos elementos fueron creados con la finalidad de añadir segmentos inline a contenidos ya existentes, sin embargo ofrecen muchas posibilidades creativas.
